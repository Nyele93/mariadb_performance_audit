Additional Innodb Buffer pool Checks

Innodb_io Capacity (the number of iops used for idle flushing and tasks like applying the change buffer operations in the background) should ideally be set to 2000. Default of 200 may be too low for workload.

The innodb_io_capacity_max (which controls how many pages per second InnoDB is allowed to flush, and roughly matches write IOPS generated by InnoDB) could be set to about 4000

The ideal value should allow the checkpoint age to be as high as possible while staying under 75% of the max checkpoint age. In general, consider Disk size and capacity when setting these values. Also monitor the outputs like InnoDB: page_cleaner: … The settings might not be optimal. As these could indicate a need to lower the above parameters’ value.

The innodb_flush_method system variable is used to define how innodb logs and data files are flushed from InnoDB. The recommended value for this parameter is: O_DIRECT. This reduces swap pressure by preventing unnecessary double buffering, which is when the operating system is buffering some data in its cache alongside InnoDB's caches.
